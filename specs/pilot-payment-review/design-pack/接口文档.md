# 接口文档

## 1. 元信息

- feature_name: payment-review-control
- design_version: design-v1.md
- req_ids: REQ-001、REQ-002、REQ-003
- 调用方: 审核端
- 被调用方: 支付管理服务

## 2. 接口清单

| 接口名 | 方法 | 路径 | 用途 | 对应 REQ-ID |
| --- | --- | --- | --- | --- |
| listPendingPayments | GET | /api/v1/payments/review/pending | 查询待审核支付单列表 | REQ-001 |
| approvePaymentReview | POST | /api/v1/payments/{paymentId}/approve | 审核通过支付单 | REQ-002 |
| rejectPaymentReview | POST | /api/v1/payments/{paymentId}/reject | 驳回支付单并记录原因 | REQ-003 |

## 3. 业务说明

- 该接口解决什么业务问题：统一支付审核端的待审核列表查询和审核流转操作
- 触发时机：审核人员进入审核列表页，或执行审核通过/驳回动作时
- 前置条件：当前用户具备审核角色，支付单状态为待审核
- 后置结果：支付单状态变更为 APPROVED 或 REJECTED，并保留审核记录

## 4. 请求说明

| 字段 | 类型 | 必填 | 含义 | 来源 |
| --- | --- | --- | --- | --- |
| paymentId | string | 是 | 支付单 ID | 路径参数 |
| reason | string | 驳回时必填 | 驳回原因 | 请求体 |

## 5. 响应说明

| 字段 | 类型 | 含义 | 备注 |
| --- | --- | --- | --- |
| paymentId | string | 支付单 ID |  |
| status | string | 支付状态 | WAIT_REVIEW / APPROVED / REJECTED |
| reviewTime | datetime | 审核时间 |  |
| reviewReason | string | 驳回原因 | 驳回时返回 |

## 6. 错误码说明

| 错误码 | HTTP 状态 | 触发条件 | 处理建议 |
| --- | --- | --- | --- |
| PAYMENT_NO_REVIEW_PERMISSION | 403 | 非审核人员访问审核接口 | 联系管理员分配审核权限 |
| PAYMENT_INVALID_STATUS | 409 | 当前状态不允许审核 | 刷新页面并重新检查状态 |
| PAYMENT_DUPLICATE_REVIEW | 409 | 支付单已审核，重复提交审核请求 | 提示用户不要重复审核 |

## 7. 依赖与时序说明

- 是否依赖其他服务：否，主要依赖支付管理内部服务和审核记录落库
- 是否涉及异步回调：否
- 是否需要幂等保护：是，审核通过和驳回都必须防止重复操作

## 8. 人工审阅关注点

- 接口命名是否清晰：是，查询、通过、驳回语义明确
- 路径设计是否与现有接口冲突：需与现有支付接口扫描结果核对
- 错误码是否和全局规范一致：需在评审时与全局错误码规范核对
