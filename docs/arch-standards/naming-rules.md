# 命名规范

## 1. 规则定位

用于统一设计文档中的类名、接口名、表名和状态机类命名，减少后续实现与 Gate 的歧义。

## 2. 必须项

- 类名使用稳定的 CamelCase 业务名词
- 表名使用 `t_` 前缀并与领域对象对应
- 状态机类命名应体现业务含义，如 `PaymentStateMachine`
- 接口路径命名应与资源语义一致

## 3. 禁止项

- 使用模糊缩写作为核心类名
- 表名与领域对象完全无关
- 状态机、Client、Repository 命名无法体现职责

## 4. 审阅关注点

- 命名是否可以直接映射到实现对象
- 同名类是否需要模块消歧
- 接口路径是否稳定且语义清晰
