# 接口设计规范

## 1. 规则定位

用于约束接口契约、接口文档和错误码语义，保证设计文档中的 API 能被后续设计包和测试直接消费。

## 2. 必须项

- 接口契约必须落盘到 OpenAPI 或等价机器文件
- 接口文档必须说明用途、前置条件、后置结果
- 请求字段和响应字段必须有最小解释
- 错误码必须和处理策略成对出现

## 3. 禁止项

- 只有路径没有用途说明
- 只有接口名没有请求/响应说明
- 只有 HTTP 状态，没有业务错误码和处理建议

## 4. 审阅关注点

- 接口路径和设计文档是否一致
- 请求/响应字段是否够支撑测试设计
- 错误码是否可操作、可排查
