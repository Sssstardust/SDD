需求：团队任务看板系统
背景
一个 10-20 人的研发团队，目前用 Excel 跟踪任务，想换成一个轻量 Web 系统。

核心功能
任务管理：创建/编辑/删除任务，字段包括标题、描述、负责人、优先级、截止日期、状态（待开始 / 进行中 / 已完成 / 已阻塞）

看板视图：按状态分列展示任务，支持拖拽改变状态

通知机制：任务被分配给某人时通知对方；任务截止前 24h 提醒负责人

评论与动态：每个任务下可以留评论，任务状态变更自动产生动态记录

权限控制：普通成员只能操作自己负责的任务，项目负责人可操作所有任务

约束条件
用户量小（同时在线不超过 50 人）
需要支持后续接入第三方（钉钉 / 飞书）通知渠道
数据不能丢，但不要求实时多端同步
